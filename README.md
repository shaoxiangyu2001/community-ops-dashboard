# 社区数据运营分析看板

一个基于真实社区帖子 Excel 数据生成的交互式 BI 看板 Demo。项目会自动识别工作表字段、清洗数据、生成分析数据集，并通过 React 页面展示社区运营洞察。

## 项目目的

原任务希望借助影刀 AI 完成社区帖子洞察和 BI 看板推荐。本项目改用 Codex 实现同样目标，包括：

- 读取并识别 Excel 中的真实字段。
- 清洗空内容、浏览量、时间、关键词和置信度。
- 提炼问题类型、情感、关注度、作者与时间趋势。
- 生成可筛选、可联动的现代化运营看板。
- 输出低置信度人工复核队列和可执行运营建议。

## 技术栈

- React 18 + Vite + TypeScript
- Tailwind CSS + 自定义数据中台样式
- Recharts
- SheetJS `xlsx`
- Lucide React

## 数据与字段识别

当前工作簿识别到 1 个数据表（`Sheet1`），表头位于第 1 行，共 200 条数据。实际匹配字段包括：

| 标准字段 | Excel 原字段 |
| --- | --- |
| 标题 | 帖子标题 |
| 链接 | 帖子链接 |
| ID | 帖子ID |
| 作者 | 作者 |
| 作者主页 | 作者个人页链接 |
| 内容 | 帖子内容（文本） |
| 发布时间 | 发布时间 |
| 浏览量 | 浏览量 |
| 数据状态 | 数据状态 |
| 问题类型 | 问题类型 |
| 情感 | 情感 |
| 关键词 | 关键词 |
| 置信度 | 置信度 |

字段匹配不是固定列号，而是通过字段别名和语义匹配完成。若换用类似结构的新工作簿，脚本会重新识别表头和字段。

## 安装与运行

建议使用 Node.js 20+ 和 pnpm。

### Windows / Codex 桌面版一键启动

如果 PowerShell 提示“无法将 pnpm 识别为 cmdlet”，直接在项目目录运行：

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\start-dashboard.ps1
```

该脚本会自动使用系统 Node/pnpm，或 Codex 桌面版自带的运行环境，并自动更新 Excel 分析数据后启动页面。

### 常规启动

```bash
pnpm install
pnpm run analyze
pnpm run dev
```

浏览器打开：

```text
http://127.0.0.1:5173
```

生产构建：

```bash
pnpm run build
pnpm run preview
```

如需分析其他工作簿：

```bash
node scripts/analyze-data.mjs "你的文件路径.xlsx"
```

脚本会更新：

- `src/data/dashboardData.json`：前端看板数据。
- `analysis-report.md`：自动生成的数据洞察摘要。

## 看板指标与模块

- 顶部指标：帖子总量、有效帖子、主要问题类型、消极反馈占比、平均浏览量。
- 全局筛选：问题类型、情感、关键词、数据状态、标题/作者/内容搜索。
- 问题类型分布：横向柱状图。
- 情感分布：环形图。
- 发帖趋势：按日面积趋势图；缺少时间字段时自动隐藏。
- 高频关键词：可点击标签云，点击后同步筛选全页面。
- 问题优先级矩阵：交叉分析讨论量、消极率与平均浏览量，并划分 P0/P1/P2。
- 类型诊断明细：展示每类问题的数量、消极率、平均浏览和高影响负面帖子。
- 具体故障场景聚类：从标题、正文与关键词识别元素捕获失败、元素偏移、脚本失效等细分场景。
- FAQ 候选：根据频次、负面程度与关注度推荐帮助文档建设顺序。
- 高影响消极案例：定位需要优先回应的高浏览量负面帖子。
- 高关注帖子：按浏览量展示有效帖子 Top 8。
- 低置信度队列：展示置信度低于 0.70 的待复核记录。
- 活跃作者：识别重复发帖和持续性需求。
- AI 核心发现与运营建议：突出可落地结论。

所有指标、图表和明细表会随筛选条件同步更新。

## 本次数据核心结论

- 总帖子 200 条，有效帖子 131 条，无效内容 69 条，无效占比 34.5%。
- “元素定位问题”最多，共 38 条，占有效帖子 29.0%。
- “Excel 与数据自动化问题”共 37 条，占 28.2%；前两类合计占 57.3%。
- 有效帖子中，中性 69 条（52.7%），消极 62 条（47.3%）；数据中没有积极标签。
- 高频关键词包括“影刀”（16 次）、“报错”（6 次）、“捕获元素”（6 次）。
- 有效帖子平均浏览量 86；高关注内容集中在元素错位、脚本稳定性和自动化工作讨论。
- 最活跃作者为 `freejanker`（5 条），其次为 `wszjtc`（4 条）。
- 发帖数据覆盖 2026-06-18 至 2026-06-23，6 月 22 日达到峰值，共 49 条有效帖子。
- 共有 13 条记录置信度低于 0.70，需要人工复核。
- 数据中未发现重复帖子 ID 或链接。

## 运营建议

1. 优先建设“元素定位问题”帮助中心，提供定位失败、元素偏移、重新捕获等标准排查流程。
2. 围绕高频关键词补充 FAQ，并在社区搜索、发帖页和客服入口前置推荐。
3. 对高浏览量消极帖子设置自动提醒和响应 SLA。
4. 用 Agent 承接元素定位、脚本稳定性、运行报错等标准化排障，并先收集环境和错误信息。
5. 建立低置信度人工复核队列，将修正标签回流到后续分类流程。

## 优雅降级策略

- 无发布时间：隐藏趋势图并展示原因。
- 无浏览量：平均浏览量显示为 `—`，隐藏浏览量排名数据。
- 无关键词：关键词模块显示空状态。
- 无情感或问题类型：使用“未分类”，避免页面报错。
- 无有效内容：保留在总量和数据状态分析中，不进入默认有效问题洞察。

## 后续优化方向
